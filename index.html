Proxy / Referer Setup

Uses corsproxy.io (free, public) to relay your stream request. The proxy forwards your request to mux.com and injects the Referer: https://www.patreon.com/ header server-side.

This is the recommended option — no setup needed, works instantly. Just paste your .m3u8 URL below and press Play.

Referer to inject

Deploy your own Cloudflare Worker for a private, reliable proxy. Free tier handles millions of requests/month.

1. Go to dash.cloudflare.com → Workers & Pages → Create Worker

2. Paste this code (replaces all existing code):

export default {
  async fetch(req) {
    const url = new URL(req.url);
    const target = url.searchParams.get('url');
    if (!target) return new Response('Missing ?url=', { status: 400 });

    const upstream = await fetch(decodeURIComponent(target), {
      headers: {
        'Referer': 'https://www.patreon.com/',
        'Origin': 'https://www.patreon.com',
        'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36',
      },
    });

    const body = await upstream.text();
    return new Response(body, {
      headers: {
        'Content-Type': upstream.headers.get('Content-Type') || 'application/octet-stream',
        'Access-Control-Allow-Origin': '*',
      },
    });
  }
};

3. Click Deploy. Your worker URL will look like https://my-worker.username.workers.dev

Referer to inject

No proxy — hls.js loads the stream directly. This will likely fail with 403 because browsers cannot set the Referer header on cross-origin XHR requests (it's a forbidden header name per the browser security spec). ModHeader also cannot override it for XHR — only for top-level navigation.

Only use this if your stream does not require a Referer check, or if you know CORS is open.
